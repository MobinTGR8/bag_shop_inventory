1 Abstract

The Bag Shop Inventory Management System is a cross-platform application built with Flutter and Supabase, designed specifically for bag retail businesses in Bangladesh. It provides a comprehensive solution for managing products (bags with type-specific attributes), inventory across multiple warehouses, sales orders with split payment support, purchase orders with receiving workflows, customer and supplier management, and detailed reporting (sales, profit, stock valuation, supplier performance).

The system implements a multi-tenant architecture where each company (shop owner) has isolated data, with role-based access control supporting Owner, Manager, Staff, and Accountant roles. An offline-first sync service with an outbox pattern ensures resilience in areas with unreliable internet connectivity. The application generates professional PDF invoices and reports, supports barcode/QR scanning for rapid product lookup, and includes CSV bulk import/export for data migration scenarios.

This project fulfills the requirements of CIT 320: Software Development Project-II at Patuakhali Science and Technology University, demonstrating mastery of modern full-stack mobile development, database design, state management, and enterprise application architecture.

2 Introduction

2.1 Project Background

Bag retail shops in Bangladesh face significant operational challenges in inventory management. Many shops still use paper-based systems or basic spreadsheets, leading to stock discrepancies, over-ordering, lost sales due to stockouts, and difficulty tracking business performance. Existing commercial inventory solutions are either too expensive for small-to-medium bag retailers or not tailored to the specific needs of a bag shop (e.g., tracking bag types, materials, sizes, and barcodes).

2.2 Problem Statement

Bag shop owners need an affordable, easy-to-use system that can:

  • Track inventory across multiple warehouses/showrooms
  • Process sales quickly at the point of sale (POS)
  • Manage purchase orders from suppliers
  • Support split payments (cash, card, mobile banking)
  • Generate insights through reports (profit, stock levels, supplier performance)
  • Work offline and sync when connectivity is restored
  • Support multiple staff members with role-based permissions

2.3 Solution Overview

The Bag Shop Inventory Management System addresses these needs with a modern, cross-platform mobile application. Built with Flutter for the frontend and Supabase (PostgreSQL) for the backend, it provides a responsive, premium UI with real-time data synchronization, comprehensive inventory tracking, a full POS workflow, and extensive reporting capabilities.

5.3 Offline-First Sync Architecture

sequenceDiagram
    participant App as Flutter App
    participant Sync as SyncService
    participant SP as SharedPrefs
    participant SB as Supabase

    App->>Sync: enqueueInsert(table, rows)
    Sync->>SP: save OutboxAction to JSON
    SP-->>Sync: OK

    Note over App: Network becomes available

    App->>Sync: syncOutbox()
    Sync->>SP: load pending actions
    SP-->>Sync: [OutboxAction...]
    
    loop for each action
        Sync->>SB: INSERT / UPDATE
        alt success
            Sync->>SP: remove action
        else duplicate
            Sync->>SP: skip & remove
        else error
            Sync-->>App: return SyncResult with error
        end
    end
    
    Sync-->>App: SyncResult(processed, remaining)

10.5 Risk Assessment

⚠️

Identified Risks

  • Internet Dependency: Full features require internet for Supabase. Mitigated by offline sync queue.
  • Data Loss: Without full local database, data could be lost if sync fails. Mitigated by outbox persistence in SharedPrefs.
  • Supabase API Changes: Supabase Flutter SDK is actively developed. Mitigated by abstracting behind repository pattern.
  • Concurrent Access: Multiple staff editing same data. Mitigated by Row Level Security and realtime subscriptions.
✅

Risk Mitigations

  • Outbox pattern with retry logic for offline operations
  • StableFingerprint for idempotency (preventing duplicate operations)
  • Backend health checker for monitoring schema integrity
  • Company-level data isolation via RLS policies
  • Comprehensive error handling with user-friendly messages
  • Supabase's built-in point-in-time recovery

12 Conclusion & Future Work

12.1 Conclusion

The Bag Shop Inventory Management System successfully delivers a comprehensive, production-ready inventory solution for bag retail businesses. Built with Flutter and Supabase, it demonstrates:

  • 40+ screens covering the complete retail workflow
  • 15+ database tables with proper relationships and Row-Level Security
  • 14 repository classes following clean architecture principles
  • Offline-first sync with outbox pattern for reliable operation
  • Professional PDF invoice and report generation
  • Premium Material 3 UI with custom brand identity
  • Role-based access control supporting multi-staff environments
